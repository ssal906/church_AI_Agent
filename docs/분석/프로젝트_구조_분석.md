# 프로젝트 구조 분석 결과

현재 `church_AI_Agent\church-platform-main` 폴더의 전체적인 구조와 소스 코드를 분석한 결과입니다.

이 프로젝트는 크게 세 가지 파트로 나뉘어진 **웹 애플리케이션(모노레포 형태)**입니다. 백엔드 API, 사용자용 프론트엔드, 관리자용 대시보드로 구성되어 있습니다.

## 1. `api` (백엔드 서버)
- **주요 기술 스택**: Node.js, Express, Sequelize (MySQL ORM)
- **특징**:
  - `express` 기반의 RESTful API 서버입니다.
  - `sequelize`를 사용하여 데이터베이스와 통신하며 DB 마이그레이션 도구를 적용하고 있습니다.
  - `jsonwebtoken`과 `passport`를 사용하여 인증(로그인 및 세션 유지)을 제어합니다.
  - `multer`와 `aws-sdk` (AWS S3)를 통해 파일/이미지 업로드를 처리합니다.
  - `node-apple-receipt-verify` 패키지를 통해 iOS 인앱 결제 검증 기능이 포함되어 있을 수 있습니다.
  - `swagger-ui-express`를 사용하여 API 문서를 제공합니다.

## 2. `front` (사용자용 웹 프론트엔드)
- **주요 기술 스택**: React, Vite, Material-UI (MUI), Recoil, React Query (TanStack Query)
- **특징**:
  - 빠르고 가벼운 번들러인 `Vite`를 기반으로 한 React 애플리케이션입니다.
  - 화면 UI는 `Material-UI` (MUI)와 `@emotion`을 혼합하여 디자인되었습니다.
  - 상태 관리를 위해 `Recoil`을 채택했으며, 서버 상태 및 데이터 페칭(Fetching) 관리는 `React Query` 및 `SWR`를 함께 사용중입니다.
  - 지도를 위한 `react-kakao-maps-sdk`, 에디터 기능을 위한 `react-quill`, 동영상 재생용 `react-player`/`react-youtube` 등이 설치되어 있습니다. (교회 및 예배 위치 안내, 주보 파일(PDF), 설교 영상 등의 기능이 있을 것으로 추정됩니다)

## 3. `admin` (관리자용 대시보드)
- **주요 기술 스택**: React, Vite, Material-UI (MUI), AG-Grid
- **특징**:
  - `front`와 유사한 기술 스택(React + Vite + MUI)을 사용하지만, 관리자 페이지 목적에 맞춰 구성되어 있습니다.
  - 방대한 데이터를 쉽게 표 형태로 보고 관리할 수 있는 강력한 데이터 그리드 라이브러리인 `ag-grid-enterprise` (및 커뮤니티 버전)가 사용되었습니다. 교인 목록, 헌금 내역, 등 다양한 관리 데이터를 처리하기 위함으로 보입니다.
  - `react-hook-form`을 활용해 복잡한 관리자용 폼 입력을 처리합니다.
